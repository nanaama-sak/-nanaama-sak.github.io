How to publish to GitHub Pages
  1. Go to github.com and create a free account (or sign in).
  2. Click the + icon → New repository. Name it yourname.github.io (use your actual GitHub username). Set it to Public. Click Create repository.
  3. On the next screen click uploading an existing file.
  4. Click Export content.json below and save the file. Then drag both blog.html and your saved content.json into the GitHub upload area. Also drag any image files you've uploaded.
  5. Rename blog.html to index.html in the file list (GitHub Pages serves index.html as the homepage). You can do this by clicking the pencil icon after upload.
  6. Scroll down, click Commit changes.
  7. Go to your repository Settings → Pages. Under Source, select Deploy from a branch, choose main, click Save.
  8. Wait ~2 minutes, then visit https://yourname.github.io — your blog is live!
  9. To update: export a new content.json from here, go back to GitHub, click the file, click the pencil icon, paste the new content, and commit. Google re-indexes within a few days.
Export Your Content

Click the button to download your current content.json. Upload this file to GitHub alongside blog.html (renamed to index.html) every time you make changes.

Images note: Images you upload are embedded as base64 data inside content.json — no separate image files needed. Large images will make the file bigger; resize photos before uploading if possible.
